# Model Selection and Evaluation

 Why Multiple Machine Learning Models Were Used
Three different classification models were trained and evaluated instead of using a single machine learning algorithm:
* Logistic Regression
* Decision Tree
* Random Forest
Different machine learning models can be compared to each other for performance on the same data set and the best model can be selected for accurate predictions. It is a commonly accepted approach in machine learning since models [learn patterns in different ways]
(https://towardsdatascience.com/why-we-train-multiple-ml-models-too-d96e106b1319). 

---

# Why Logistic Regression?

Logistic Regression was used as the **baseline model** because it is one of the [most popular](https://en.wikipedia.org/wiki/Logistic_regression) machine learning models for binary classification problems.
Logistic Regression is very suitable for our problem because the target variable (Diabetes) has only positive and negative classes.

### Advantages
* Fast to train
* Easy to interpret
* Works well when the relationship between features and the target is approximately linear
* Probability estimates can be predicted

### Limitations
* Assumes a linear relationship between the input features and the log-odds of the target variable.
* May not be able to capture the complex relationship between the medical symptoms. 
Therefore, Logistic Regression can be used as a benchmark for more complex models. 

---

# Why Decision Tree?

Decision Trees were chosen as they can learn non-linear decision boundaries and are easy to understand.
Unlike Logistic Regression, a Decision Tree performs numerous splits over the dataset based on feature values on why a decision can be made, making it possible to learn more complicated relationships such as between the symptoms and diabetes.

### Strengths
* It's easy to interpret
* Works with both categorical and continuous features
* Learns non-linear decision boundaries
* Doesn't require much data preprocessing

### Weaknesses
* Often overfits to the training set
* Consistency can drop dramatically with a small change in the dataset
* Often less accurate than ensemble learners
Although easy to interpret, Decision Trees are sometimes very simple and less powerful, due to overfitting.

---

# Why Random Forest?

Random Forest is an ensemble learning algorithm that uses a combination of many Decision Trees instead of one.
Every tree is trained on a different random subset of the training data. The final prediction is obtained by aggregating the predictions of all the trees, typically through majority voting.
This leads to better generalization and reduced overfitting.

### Strengths
* It's very accurate
* Reduces overfitting compared to a single Decision Tree
* Learns non-linear decision boundaries
* Robust to noisy data
* Estimation of feature importance

Because most medical datasets exhibit complex relationships between the symptoms, Random Forest is particularly good at predicting the disease.

# Why Is Accuracy Not Enough?

Although many people report accuracy for the model, it is enough to mislead the audience in some cases. This might happen with medical datasets.
For example, if most of the patients are diabetic, a model can say that every patient is diabetic, and so it predicts a high accuracy, while it totally fails at identifying non-diabetic patients.
So we used other evaluation metrics besides accuracy.

---

# Accuracy

Accuracy estimates the percentage of correctly classified samples.
Formula
Accuracy = (TP + TN) / (TP + TN + FP + FN)
Where
TP = True Positives
TN = True Negatives
FP = False Positives
FN = False Negatives
The higher the accuracy, the better the model performs in overall prediction.

---

# Precision

Precision estimates how many of the patients predicted as diabetic are actually diabetic.
Formula
Precision = TP / (TP + FP)
The higher the precision, the more likely the model produced a small number of false positive predictions.
This is a required metric in situations where we try to limit unnecessary treatments or medical tests.
 
---

# Recall (Sensitivity)

Recall estimates how many of the actual diabetic patients were identified by the model.
Formula
Recall = TP / (TP + FN)
Recall is critically important in a healthcare setting where missing a patient's diagnosis can postpone the diagnosis and treatment of a disease.
A higher recall rating means less diabetic patients were missed.

---

# F1-Score

F1 combines the F1 score of Precision and Recall.
Formula
F1 = 2 * (Precision * Recall) / (Precision + Recall)
F1 is recommended when the number of false positives and false negatives are both important.
A higher F1-score means the number of diabetic patients are being better identified and the number of false predictions is much smaller.

---
 
# ROC-AUC Score  

A Receiver Operating Characteristic - Area Under the Curve (ROC-AUC) measures the ability of a model to discriminate between diabetic and non-diabetic patients at various threshold levels.
A value close to **1.0** represents good discrimination.

The general classification:
* 0.50 → Random
* 0.60-0.70 → Poor
* 0.70-0.80 → Fair
* 0.80-0.90 → Good
* 0.90+ → Excellent

The Random Forest model achieved a ROC-AUC of **96.79%**, showing excellent predictive power.
  
---

# Why SHAP
                   
The machine learning models such as Random Forest usually offers high accuracy but not the reason why a certain patient was classified into a specific class.
As a solution to the black box problem, SHAP (SHapley Additive exPlanations) was used to explain the final Random Forest model.
SHAP provides a contribution score to each feature for each individual prediction, and we can identify and explain which features of a patient are most important for the model's decision.
Thus, the model is able to provide both the prediction and the explanation for better decision making in the real healthcare settings.
