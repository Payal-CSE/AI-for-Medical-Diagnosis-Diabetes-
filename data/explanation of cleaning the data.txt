# explanation of cleaning the data just in case u didnot understand any step

The diabetes dataset had to be cleaned and preprocessed before we could use it to build machine learning models. This step is really important because it makes sure the data is accurate and consistent. If we do not do this our models might not work well or might even give us wrong results.



# Step 1: Import Required Libraries

import pandas as pd
import numpy as np

We use two Python libraries for this project:

Pandas is used to load and manipulate data.
NumPy is used for computing.


# Step 2: Load the Dataset
df = pd.read_csv("diabetes_data_not_cleaned.csv")

We load the diabetes dataset into a Pandas DataFrame.
We check to make sure the dataset was loaded successfully and look at the number of rows and columns.
This is important to do before we start preprocessing the data.


# Step 3: Rename Columns
The original dataset has column names that re not very clean.
We rename each column to make it easier to read and understand.

For example:

| Original           | Renamed          |
| ------------------ | ---------------- |
| sudden weight loss | SuddenWeightLoss |
| visual blurring    | VisualBlurring   |
| delayed healing    | DelayedHealing   |

Renaming columns is helpful because it makes our code easier to read and understand.
It also helps prevent problems that can happen when there are spaces in column names.
It makes all the feature names consistent throughout the project.


# Step 4: Dataset Inspection
Before we start cleaning the data we do checks:

df.info()
df.isnull().sum()
df.duplicated().sum()

These checks give us information about the dataset like the number of observations the type of data in each feature and if there are any missing values.
Understanding the dataset before we preprocess it helps us find any quality issues.

# Step 5: Remove Duplicate Records

df = df.drop_duplicates()

We remove any duplicate records from the dataset.
Duplicate records can be a problem because they can give much importance to repeated samples.
By removing duplicates we make sure each patient record appears once in the dataset.


# Step 6: Clean Text Columns
We select the categorical or text columns.
For every text column we convert the values to string format remove any leading or trailing spaces and convert the text to Title Case.

For example:

| Before | After |
| ------ | ----- |
| yes    | Yes   |
| YES    | Yes   |
| yes    | Yes   |
| male   | Male  |

This standardization helps prevent encoding errors that can happen when the formatting is not consistent.


# Step 7: Verify Values

We display the unique values of every categorical column.
This validation step helps us confirm that only the expected categories are present there are no spelling mistakes. There are no unexpected values in the dataset.

For example:

Polyuria
['Yes 'No']'

This verification is important to make sure that categorical encoding will work correctly.



# Step 8: Encode Binary Features

We convert the following medical symptoms from text into numerical values:
* Polyuria
* Polydipsia
* SuddenWeightLoss
* Weakness
* Polyphagia
* GenitalThrush
* VisualBlurring
* Itching
* Irritability
* DelayedHealing
* PartialParesis
* MuscleStiffness
* Alopecia
* Obesity

The encoding scheme is:

Original | Encoded |

| Original | Encoded |
| -------- | ------- |
| Yes      | 1       |
| No       | 0       |
This conversion is essential because machine learning algorithms need numerical input.

# Step 9: Encode Gender

We convert the gender feature into values:

| Original | Encoded |
| -------- | ------- |
| Male     | 1       |
| Female   | 0       |

This allows the feature to be processed by machine learning algorithms.


# Step 10: Encode the Target Variable

The target variable, **Diabetes** represents whether a patient has diabetes or not.
The encoding scheme is:

| Original | Encoded |
| -------- | ------- |
| Positive | 1       |
| Negative | 0       |

After encoding the dataset is ready for binary classification.

# Step 11: Convert Data Types
We convert all encoded features to numeric data types:

pd.to_numeric(errors="coerce")

This ensures numerical consistency, compatibility with machine learning models and detection of invalid values by converting them to `NaN`.


# Step 12: Final Validation

We perform final quality checks.
The notebook verifies missing values, data types, the first five records and dataset dimensions.
These checks confirm that preprocessing was completed successfully and no errors were introduced.


# Step 13: Save the Clean Dataset
We export the cleaned dataset as:

diabetes_cleaned.csv


This processed dataset is now ready for:

* Exploratory Data Analysis (EDA)
* Feature Selection
* Machine Learning Model Development
* Model Evaluation
* AI (SHAP)
* Research Paper Experiments
